## はじめに

「今さらClaude Code入門？」と思う方も多いかもしれません。
でも、これからさわり始めるアーリーマジョリティのために、敢えて今この記事を出そうと思います。
この記事では、Claude Codeとは何か、どう始めるかを書きます。

## Claude Code の概要

Claude Code (クロード・コード)は、ターミナル上で対話しながら作業を進める開発支援ツールです。
GitHub Copilotのような単発のコード提案とは違い、調査・修正・検証・変更内容の取りまとめを、ひとつのセッション内で連続して扱えます。

![](/images/claude_demo.gif)

## ChatGPT／GitHub Copilot との役割分担

AIを使った開発ではChatGPTやGitHub Copilotなども良く使われます。
Claude Codeは同じ「AIを使った開発支援」でも、得意な領域が違います。

- `ChatGPT`：設計検討、仕様の言語化、アプローチの比較、レビューや観点整理
- `GitHub Copilot`：IDE内の入力補助（コード補完、スニペット作成などの局所的な実装支援）
- `Claude Code`：リポジトリ内の調査→修正→検証→変更内容の取りまとめを一連の作業として進める

Copilotが「コードを書く」ことに強いのに対して、Claude Codeは「作業を進める」ことに向いています。
IDEで記述し、ターミナル側で調査・実行・差分整理を担当する、という分担が作りやすい組み合わせです。

## 対応できる作業領域

Claude Codeをコード生成の道具としてのみ捉えると、できることを見誤ります。
調査、検証、原因切り分け、変更内容の取りまとめなど、実装以外の工程も対象として扱えます。

こうした作業をうまく任せられるかは、「次に何をするか、何をもって成功とするか」を言語化できるかどうかにかかっています。
Claude CodeはターミナルベースのCLIツールです。
ターミナル中心の開発スタイルに慣れていれば、その経験が活きてきます。

## 導入

### インストール

ターミナルで以下を実行します。
*※ npmインストールは現在非推奨です。*

#### Mac
```bash
curl -fsSL https://claude.ai/install.sh | bash
```

#### Windows
```bash
# Windows PowerShell:
irm https://claude.ai/install.ps1 | iex

# WinGet (Windows):
winget install Anthropic.ClaudeCode
```

自動更新されるので、常に最新版が使えます。

その他の方法（Homebrew など）は公式ドキュメントを参照してください。
[公式ドキュメント - インストール](https://code.claude.com/docs/ja/setup)

### 起動

対象リポジトリのディレクトリで `claude` を実行します。

```bash
claude
```

初回は認証の設定が求められます。

## まず試すのに良い作業

最初は、結果がわかりやすく、差分が過剰に広がりにくい題材が適しています。

### 1. リポジトリの把握

例として以下のような指示が考えられます。

- 目的、主要ディレクトリ、起動方法、テスト方法を要約する
- 不明点は推測で埋めず、質問として列挙する

この作業は Plan mode（後述）で始めると、意図しない変更を防げます。

### 2. テスト失敗の修正

例として以下のような指示が考えられます。

- テストを実行し、失敗ケースを特定する
- 原因を簡潔に説明する
- 修正後に再実行し、通過を確認する
- 変更点を箇条書きで要約する（コミットメッセージ案を含む）

成否がテスト結果として明確に判定できるため、ツールの動作確認に向いています。

### 3. 影響範囲の確認を伴う小変更

例として以下のような指示が考えられます。

- 関数名や設定キーの変更に伴う参照箇所を列挙する
- 修正方針（順序、確認手順、注意点）を提示する
- 不要変更の除去、コミット単位の分割、説明文の作成

## 権限（permission）と安全設計

Claude Code は、無制限に任意コマンドを実行する設計ではありません。操作に対して許可を挟む仕組みがあり、許可モードを切り替えながら段階的に作業を進めるのが現実的です。

### Plan mode

Plan mode は、調査と計画作成に限定したモードです。ファイル変更やコマンド実行はせず、方針を固める段階で使います。

```bash
claude --permission-mode plan
```

セッション中のモード切替は Shift+Tab で行えます（Auto Accept　/ Plan / Normal の順に切り替わります）。
![](/images/claude_mode_change.gif)

### 許可設定の方針

セッション中に `/config` を実行すると、権限設定を確認・変更できます。設定はグローバルとプロジェクト単位で分かれています。

認証情報へのアクセスなどの危険な操作を明示的に拒否し、低リスクな操作だけを許可する方針がおすすめです。

設定例を以下に示します。

- テスト実行、lint 実行 → 許可(allow)
- 認証情報へのアクセス → 拒否(deny)
- rmやmvなどの破壊的操作 → 都度確認(ask)

## 次に読む対象（発展）

基本的な使い方（調査→修正→検証→取りまとめ）に慣れてきたら、さらに以下のトピックを学ぶことをおすすめします。

### MCP（外部連携）

外部のツールやデータソース（チケット管理、監視システム、DB、社内 API など）を Claude Code などから接続するため仕組みです。コードだけでは完結しない作業が多い環境で有効です。

### Skills（定型化）

レビュー観点、PR本文のテンプレート、ログ調査の手順などを再利用可能な単位として定義し、毎回の指示を省略できるようにします。CLAUDE.md に書いた観点を Skills 化する、といった使い方が典型です。

### Hooks（イベント駆動の自動化）

特定のタイミング（セッション開始時、コミット前など）でチェックや処理を自動で実行し、手順の抜け漏れを防ぎます。
lintの自動実行やコミットメッセージの検証などが代表的な用途です。

### Subagents（役割分担）

調査役、修正役、レビュー役のように役割を分けて作業を進めることで、関心の分散を抑えます。長い作業や探索範囲が広い作業で、見通しを改善する方向に有効です。

## まとめ

Claude Code は、ターミナルベースでリポジトリを扱い、調査・修正・検証・変更内容の取りまとめを一連の工程として進めやすいツールです。最初は「結果がわかりやすい題材」から入り、Plan mode で許可範囲を限定しながら段階的に委譲範囲を広げるのが堅実な進め方です。
