## はじめに

「今さらClaude Code入門？」と思う方も多いかもしれません。
でも、これからさわり始めるアーリーマジョリティのために、敢えて今この記事を出そうと思います。
この記事では、Claude Codeの位置づけと最初の一歩について書きます。

## Claude Code の概要

Claude Code (クロード・コード)は、ターミナル上で対話しながら作業を進める開発支援ツールです。
単発のコード提案ではなく、調査・修正・検証・変更内容の取りまとめを、ひとつのセッション内で連続して扱える点が特徴です。

## ChatGPT／GitHub Copilot との役割分担

AIを使った開発ではChatGPTやGitHub Copilotなども良く使われます。
Claude Codeは、同じ「AI を使った開発支援」でも、向いている作業が異なります。

- ChatGPT：設計検討、仕様の言語化、アプローチの比較、レビューや観点整理
- GitHub Copilot：IDE内の入力補助（コード補完、スニペット作成などの局所的な実装支援）
- Claude Code：リポジトリ内の調査→修正→検証→変更内容の取りまとめを一連の作業として進める

Copilot が「記述の支援」に強いのに対して、Claude Code は「開発工程の進行」に適しています。
IDEで記述し、ターミナル側で調査・実行・差分整理を担当する、という分担が作りやすい組み合わせです。

## 対応できる作業領域

Claude Code をコード生成の道具としてのみ捉えると、対象範囲を過小評価することになります。開発では実装以外の工程に工数がかかることが多く、Claude Code はそこも対象として扱えます。

- 調査：参照箇所の探索、影響範囲の洗い出し、既存仕様の把握
- 検証：ビルド、テスト、lint、型チェックの実行と結果の解釈
- 原因切り分け：ログ／スタックトレースの要点整理、再現条件の明確化
- 変更内容の取りまとめ：不要変更の除去、コミット単位の分割、説明文（コミットメッセージ／PR本文）の作成

この範囲を活かせるかどうかは、モデルの能力以前に「次に何を実行し、何で成否を判定するか」を指示として定義できるかに依存します。ターミナル上の作業に慣れているほど、その指示が具体的になります。

## 導入

### インストール

npm 経由でインストールします。

```bash
npm install -g @anthropic-ai/claude-code
```

Node.js 18 以上が必要です。その他の方法（Homebrew など）は公式ドキュメントを参照してください。

### 起動

対象リポジトリのディレクトリで `claude` を実行します。

```bash
cd <リポジトリのパス>
claude
```

初回は認証の設定が求められます。

## 初回に適した作業

初回は、結果が観測しやすく、差分が過剰に広がりにくい題材が適しています。

### 1. リポジトリ把握（読み取り中心）

依頼例：

- 目的、主要ディレクトリ、起動方法、テスト方法を要約する
- 不明点は推測で埋めず、質問として列挙する

この作業は Plan mode（後述）で始めると、意図しない変更を防げます。

### 2. テスト失敗の修正（短い反復）

依頼例：

- テストを実行し、失敗ケースを特定する
- 原因を簡潔に説明する
- 修正後に再実行し、通過を確認する
- 変更点を箇条書きで要約する（コミットメッセージ案を含む）

成否がテスト結果として明確に判定できるため、ツールの動作確認に向いています。

### 3. 影響範囲の確認を伴う小変更

依頼例：

- 関数名や設定キーの変更に伴う参照箇所を列挙する
- 修正方針（順序、確認手順、注意点）を提示する
- 不要変更の除去、コミット単位の分割、説明文の作成を行う

## 権限（permission）と安全設計

Claude Code は、無制限に任意コマンドを実行する設計ではありません。操作に対して許可を挟む仕組みがあり、許可モードを切り替えながら段階的に作業を進めるのが現実的です。

### Plan mode

Plan mode は、調査と計画作成に限定したモードです。ファイル変更やコマンド実行を行わず、方針を固める段階で使います。

```bash
claude --permission-mode plan
```

セッション中のモード切替は Shift+Tab で行えます（Plan / Auto Accept / Normal の順に切り替わります）。

### 許可設定の方針

REPL 上で `/config` を実行すると、許可設定を確認・変更できます。設定はグローバルとプロジェクト単位で分かれています。

許可を広く与えるより、危険な操作を明示的に拒否し、低リスクな操作だけ許可する方針が安定します。

設定例：

- テスト実行、lint 実行 → 自動許可
- 外部通信を伴う操作、認証情報へのアクセス → 拒否
- 削除や広範囲の一括置換 → 都度確認

## 次に読む対象（発展）

基本の流れ（調査→修正→検証→取りまとめ）が安定した後に、拡張として検討しやすい領域です。

### MCP（外部連携）

外部のツールやデータソース（チケット管理、監視システム、DB、社内 API など）を Claude Code に接続する枠組みです。コードだけでは完結しない作業が多い環境で有効です。

### Skills（定型化）

レビュー観点、PR本文のテンプレート、ログ調査の手順などを再利用可能な単位として定義し、毎回の指示を省略できるようにします。CLAUDE.md に書いた観点を Skills 化する、といった使い方が典型です。

### Hooks（イベント駆動の自動化）

特定のタイミング（セッション開始時、コミット前など）でチェックや処理を自動実行し、手順の抜け漏れを防ぎます。lint の自動実行やコミットメッセージの検証などが代表的な用途です。

### Subagents（役割分担）

調査役、修正役、レビュー役のように役割を分けて作業を進めることで、関心の分散を抑えます。長い作業や探索範囲が広い作業で、見通しを改善する方向に有効です。

## まとめ

Claude Code は、ターミナルベースでリポジトリを扱い、調査・修正・検証・変更内容の取りまとめを一連の工程として進めやすいツールです。初回は「結果が観測できる題材」から入り、Plan mode で許可範囲を限定しながら段階的に委譲範囲を広げるのが堅実な進め方です。
