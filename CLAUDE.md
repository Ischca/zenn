# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## リポジトリ概要

Zenn（https://zenn.dev）に公開する技術記事・書籍を管理するリポジトリ。

## コマンド

```bash
npm run preview      # ローカルプレビュー（http://localhost:8000）
npm run new:article  # 新規記事作成
npm run lint         # textlintで記事をチェック
npm run lint:fix     # textlintで自動修正
```

## 構成

- `articles/` - 記事（Markdown）
- `books/` - 書籍
- `images/` - 画像

## 文体ガイドライン

### 冗長表現の削除

```
× することができます → ○ できます
× 実行を行う → ○ 実行する
× 〜する必要があります → ○ 〜します
× 〜ということ → ○ 〜こと
× 言葉にする → ○ 言語化する
```

### 助詞の重複を避ける

```
× 結果が変わることがあります → ○ 結果が変わる場合があります / 結果は変わることがあります
× 次工程に進む前に → ○ 次工程へ進む前に
× 影響が出る可能性があります → ○ 影響する可能性があります
```

### AI生成っぽいリスト形式を避ける

```markdown
× - **調査**：参照箇所の探索、影響範囲の洗い出し
× - 調査：参照箇所の探索、影響範囲の洗い出し（詳細すぎる）

○ - 調査では、参照箇所の探索や影響範囲の洗い出しを行える
○ 調査、検証、原因切り分けなど（シンプルに列挙）
```

Getting Started記事のような導入記事では、詳細な列挙より概要を伝えることを優先する。

### 見出しの使い方

ラベル的なテキストは見出しにする。

```markdown
× 悪い例
  ```code```

○ #### 悪い例
  ```code```
```

### 改行スタイル

Zenn向けに改行を多めに入れる。機械的に「。」で改行するのではなく、文脈で判断する。

- 1文ごとに改行を入れる
- トピックの切れ目で改行
- 長い文（40文字超）は分割を検討

### ターゲット読者への配慮

ターミナル中心の開発スタイルに慣れた人が主な読者。ただし、初心者を振り落とさない表現にする。

```
× ターミナルを主戦場にしている人（不自然）
× シェルやgit操作に習熟していれば（gitは常識なので言及不要）
× 基礎知識が前提になります（甘く見ている印象）

○ ターミナル中心の開発スタイルに慣れていれば、その経験が活きてきます
```

「武器になる」「活きてくる」のように、持っていればプラスになるニュアンスで書く。

### 許容される表現

- 導入部での柔らかいトーン（「かも」「思う」）は意図的に残す
- レトリカルな疑問での「？」
- 引用文内の表現は変更しない
